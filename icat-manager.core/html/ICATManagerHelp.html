ICAT Manager Help

ICAT Manager is a standalone Java client for visualizing and managing ICAT instances.

Features

  • Works for any version of ICAT starting from version 4.2.0 thanks to a JAX-WS dynamic client.
  • Allows display, editing and creation of any ICAT entity (based on your access rights).
  • Configurable pagination, adapted to large instances. Sorting and filtering entities on any field.
  • Remote update: install once, update from the interface (and no need to update the software when you update your ICAT).

Prerequisites Back to top

For running this program you need a JDK 7 installed (tested only with Oracle JDK 1.7, which you can download here http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html).

Installation and launch Back to top

Download the archive for your platform at http://ftp.esrf.fr/pub/bliss/icatmanager/.

Unzip / untar the archive. It will create a folder named 'icatmanager' that contains the complete software.

ICAT Manager needs a JDK to run (as it generates the client classes on-the-fly), in order to do so you need to:

  • On linux, either:
    • Set the JAVA_HOME environment variable to point to a JDK and execute ./IcatManager from the icatmanager folder to start the program.
    • Execute ./IcatManager -vm PATH_TO_JDK/bin/java from the icatmanager folder to start the program.
    • Edit the IcatManager.ini file (see below for details). Execute ./IcatManager from the icatmanager folder to start the program.
  • On Windows, either:
    • Create a shortcut to the IcatManager.exe file, edit its properties and add to the 'Target' field -vm "PATH_TO_JDK\bin\javaw.exe" (with the quotes). Use the shortcut to start the program.
    • From the console, execute from the icatmanager folder IcatManager -vm "PATH_TO_JDK\bin\javaw.exe" (with the quotes) to start the program.
    • Edit the IcatManager.ini file (see below for details). Then double-click directly on the executable to start the program (or call it from the command line).
  • On MacOS you will need to edit the IcatManager.app/Contents/MacOS/IcatManager.ini file (see below for details). Then double-click on the application to start the program.

Setting JVM options Back to top

For setting options to the JVM used by IcatManager, you have to edit the IcatManager.ini file. You can also configure the JDK to use by this method on linux and Windows, and only by this method on MacOS.

Details on this operation and the available options can be found on the eclipse wiki.

Pay attention to the order of the options in the ini file, this is critical for this to work.

Proxy configuration Back to top

If you are behind a proxy, add the following lines at the end of the IcatManager.ini file (replace the example values with the correct settings for your location):

-Dicatmanager.http.proxyHost=proxy.url
-Dicatmanager.http.proxyPort=1234
-Dicatmanager.https.proxyHost=proxy.url
-Dicatmanager.https.proxyPort=1234
-Dicatmanager.http.nonProxyHosts=localhost|*.local

Defining a new ICAT server Back to top

Add an ICAT server by using the Server → New server menu or the New server button on the left panel, you will then be prompted for the server URL. You can set either the service, wsdl or server URL.

For example these are all valid inputs:

For ESRF ICAT: For ISIS ICAT:
https://wwws.esrf.fr/icat https://icatisis.esc.rl.ac.uk/
https://wwws.esrf.fr/icat/ICATService https://icatisis.esc.rl.ac.uk/ICATService/
https://wwws.esrf.fr/icat/ICATService/ICAT?wsdl https://icatisis.esc.rl.ac.uk/ICATService/ICAT?wsdl

You can define as many ICAT servers as you want.

The status of each server defined in the list is shown with an icon next to its name: means there is no active connection to this server, means the software is connected to this ICAT server, and means the software attempted to connect but failed.

To delete an ICAT server from the list of servers, use the Server → Delete server menu or the Delete button .

Connecting to an ICAT instance Back to top

To connect to the ICAT server you defined, double-click on it in the list of servers, or select it and use the Server → Connect server menu or the Connect server button . You will be prompted for an authentication method, a user name and a password.

The authentication method to use depends on the configuration of the ICAT, typical authentication methods include 'db', 'ldap', 'esrf' (ESRF custom authenticator ), 'uows' (ISIS custom authenticator). In case of doubt ask your local ICAT administrator.

The user name and password are the ones you usually use to connect to ICAT. If you use the anonymous ('anon') authentication leave both fields empty. Note that the software only displays the content you have access to according to the credentials you use.

Once connected the version of the ICAT server is shown to the right of its URL.

To disconnect from an ICAT server, use the Server → Close connection menu or the Disconnect button .

Displaying entities Back to top

Once connected to the ICAT server, a list of available entities (Dataset, Datafile, etc.) can be displayed in the left panel. To show or hide the list, double-click on the server or click the little arrow before the server URL. Select an entity and use the Entities → Open entity menu or the Open entity button . This will open a new view in the right panel showing the accessible entities of that type.

A new view will be opened for each type of entity on a given server. The server and the entity name are displayed in the tab.

Be careful when you access several ICAT servers at the same time and check the tab name to make sure you are interacting with the right server.

Counting entities Back to top

You can display the number of entities of a given type in ICAT by selecting an entity in the left panel and using the Entities → Count entities menu. The count will be displayed next to the entity name.

Note that this command counts the number of entities available to the user, not all the entities in the database. Counting entities can be slow on large instances, this is why this value is never updated automatically.

Display options, sorting and filtering Back to top

By default, entities are displayed by page of 50 and sorted by descending ids (latest created first). You can change the page size by using the combo box at the top of the view, next to the First / Previous / Next buttons. On the top left of the view you will see the entity name and the number and indices of the displayed entities (e.g. 'Dataset from 1 to 50').

By default the number of available entities is not looked up. If you want to know how many entities are in the current list, use the 'Keep count' button . This will display the total number of entities after the current indices.

Note that the total number of entities is updated at each query so activating this option may slow down operations.

You can sort the entities by clicking on column headers. Click a second time on the header to reverse the sort order. The column used for sorting is flagged with a sort marker in the column header, either or depending on the sort order.

If you sort by an associated entity (e.g. sorting datafiles by their associated dataset), the sorting is done on the entity ids by default. If you want to sort by entity names, use the Sort by name button .

You can filter the list of entities by entering a valid query in the 'Filter' box using the Concise syntax. For details on the query syntax please refer to the ICAT manual http://icatproject.org/mvn/site/icat/client/4.3.3/manual.html. To clear the query, use the Clear query button .

Each time you change the display options of the filtering a new query will be made to the ICAT server. If you want the refresh the displayed data use the Refresh button or the Entities → Refresh menu.

Note that columns can be reordered by dragging their header and this ordering is kept between sessions.

Displaying related entities Back to top

When you right-click on an entity or a selection of several entities, a context menu is displayed. In addition to the usual commands available on entitites (New, Edit, ...) there are also two submenus:

"Show corresponding ...". This submenu contains a list of entities associated with the selected one(s). By selecting an entity in this submenu you will be shown the entries of that type in ICAT that relate to the selected entity or entities.

"Show entities with same...". This submenu contains the list of fields for the selected entity or entities. By selecting a field in this submenu you will be shown all entities of the same type than the selected one(s) that share the same value in that field.

Editing entities Back to top

If you select a line in the right panel, you can edit the selected entity by using the Entities → Edit entity menu or the Edit button . This will open the Entity editing dialog that allows to modify any field. If the field is a character or numeric value you can type the value in the field. Booleans are represented by check boxes (selected means True) and predefined lists by combo boxes.

Association fields (e.g. the Dataset associated to a Datafile) behave in a similar way than predefined lists, except that they display a limited number of entities in the combo box. By default the 50 entities created recently are displayed, but you change the list by typing a part of the name (or full name) of the entity.

For example, if you edit a datafile, the dialog box only display the 50 most recent datasets in the combo box. Start typing a part of the name of the dataset you are looking for, and a list of matching datasets will be displayed. Note that the list is always limited to 50 items and the search is case-sensitive on ICAT before 4.3.3 and case-insensitive for more recent versions.

For entities that also have a full name (e.g. Facilities) you can type part of the full name too. For entities that have neither a name or a full name you have to use the full id.

Changes are reflected in ICAT only when the user click OK. Using Cancel to close the dialog ensures no modification will be made to the entities.

Editing multiple entities Back to top

If you select several lines in the right panel, you can edit all the selected entities at once by using the same Entities → Edit entity menu or the Edit button . This will open the same Entity editing dialog but with several differences:

  • For fields where all entities in the selection share the same value, the value is displayed (see above for display details).

  • For fields where none of the entities in the selection has a value set, no value is displayed.

  • For fields where entities do not share the same value, no value is displayed, the control is disabled (greyed out) and a multiple values flag is shown.

  • Changing the value of a field sets the new value on all selected entities. Modifying fields where entities do not share a common value is disabled by default. To enable editing those fields, user must tick the check box next to the multiple values icon . Unticking the check box again prevents any modification to the field, even if the value in the control was modified.

Changes are reflected in ICAT only when the user click OK. Using Cancel to close the dialog ensures no modification will be made to the entities.

Disabled fields are guaranteed to not change on any entity in the selection: even if you changed the value and clicked OK, if the field is disabled at that moment no value is changed for that field.

In place editing Back to top

Alternatively, you can use in-place editing in the right panel. For activating the feature, click on the Toggle in-place editing button in the right panel. Note that in-place editing does not allow to edit association fields for the moment.

Please be careful with this feature as all modifications are immediately sent to the ICAT server and there is no undo.

Adding entities Back to top

You can add an entity by using the Entities → New entity menu or the New entity button . When you do so, the Entity editing dialog is shown, but all fields are empty.

Note that there is no verification that the creation will be successful, i.e. the software does not check the mandatory fields are filled nor that you have the right to create the entity.

Deleting entities Back to top

When you select one or several lines from the right panel, you can use the Entities → Delete entity menu or the Delete button to delete them. Please keep in mind that the delete method of ICAT propagates to dependent entities (cascade delete), so for example deleting a facility will remove everything related to this facility recursively, i.e. the entire catalogue.

Updating the software Back to top

Updates of the software will be announced on the icatgroup mailing lists.

You can use the Help → Check for updates … menu to check for updates of the software. If it finds one (and it should if it has been announced and you configured your proxy correctly) you will be prompted for confirmation and then, if you accept the update, for restarting the software after the update.

Support Back to top

Most errors are traced in the log\icat_manager.log file in the installation folder. In particular all queries to ICAT are logged, so if something does not work as you think it should, first check the log.

Here are a few operations that can help you:

  • If your interface is scrambled, you can use the Help → Reset interface menu to restore the interface to its default (necessitates a restart).

  • If the program refuses to start, or if the list of servers is broken, you can try to delete the .icat_servers file from your home folder and see if the program starts.

Support requests can be sent to christophe.cleva -/at/- esrf.fr.

Known issues Back to top

  • Due to a bug in the Eclipse platform, if you close the right panel when it is empty it will disappear from the application interface. Until a workaround is found or the platform is updated, you can use the Help → Reset interface menu to restore the interface (necessitates a restart).

  • On linux, the embedded browser does not allow to access external links when behind a proxy

  • This software has never been tested on MacOS
